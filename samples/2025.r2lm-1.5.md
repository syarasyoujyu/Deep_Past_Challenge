# Evaluating the Performance of Transformers in Translating Low-Resource Languages through Akkadian

## 論文の概要

- タイトル: Evaluating the Performance of Transformers in Translating Low-Resource Languages through Akkadian
- 著者: Daniel Jones, Ruslan Mitkov
- 会議: Proceedings of the First Workshop on Comparative Performance Evaluation: From Rules to Language Models, RANLP 2025
- 対象課題: アッカド語のローマ字転写文を英語へ機械翻訳すること

この論文は、低資源言語の機械翻訳をどう改善できるかを、アッカド語を題材に検証した研究である。アッカド語自体は現代では使われない古代言語だが、ORACC や CDLI のようなデジタル化プロジェクトにより、楔形文字をラテン文字へ転写したテキストと英訳の対応データが比較的豊富に存在する。そのため、低資源言語でありながら、実験用ベンチマークとしてはかなり恵まれた条件を持つ。

論文の中心的な問いは次の通りである。

- 低資源言語翻訳では、どの Transformer 系モデルが有効か
- encoder-decoder 型の NMT と decoder-only 型 LLM のどちらが有利か
- 関連言語で事前学習されたモデルの転移が有効か
- 翻訳品質と推論速度のトレードオフはどうなるか

この論文の結論を先に言うと、総合的にはアラビア語事前学習の MarianMT が最もバランスが良く、BLEU だけなら Mistral-7B が最良、短文では Qwen や Mistral のような LLM が強い、という結果である。

## 何を目的としている論文か

この研究の目的は、単にアッカド語を訳す高精度モデルを作ることだけではない。より大きな目的は、低資源言語向け翻訳システムを設計する際の知見を得ることにある。

具体的には次を明らかにしようとしている。

- 低資源条件でも既存の Transformer を fine-tuning すればどこまで戦えるか
- パラメータ数の大きい LLM が常に有利とは限らないのではないか
- 言語的に近い言語で学習済みの翻訳モデルを転用すると有利ではないか
- 学術用途や実運用では、精度だけでなく速度も重要ではないか

論文では、アッカド語はデータ量の少ない言語一般の代理として扱われている。ただし著者たちも注意しているように、アッカド語は「低資源言語ではあるが、注釈付き並列データが比較的豊富」という特殊事情がある。そのため、他の本当にデータが乏しい言語へそのまま一般化するには注意が必要だとしている。

## 入力と出力は何か

この論文の翻訳タスクはかなり明確で、入出力は以下である。

- 入力: 楔形文字そのものではなく、ラテン文字に転写されたアッカド語テキスト
- 出力: その英語訳

重要なのは、画像認識や楔形文字 OCR は扱っていない点である。つまり、

- 入力はタブレット画像ではない
- 入力はすでに人手または既存資源によって転写済みのテキスト
- 出力は自由生成される英語文

したがって研究対象は、厳密には「楔形文字から英語へ」ではなく、

`Akkadian transliteration -> English translation`

である。

論文中の例では、天文文書や王碑文のようなテキストを入力し、それに対して英訳を生成している。つまり、モデルの出力は分類ラベルでもタグ列でもなく、自然言語文そのものとなる。

## なぜアッカド語を使うのか

著者たちは、アッカド語を低資源翻訳研究の実験台として使う理由をいくつか挙げている。

- 現代では使われないため低資源言語に分類される
- ORACC や CDLI により比較的大規模なデジタル資源が整備されている
- 原文は楔形文字だが、実験ではラテン文字転写を使えるため NLP に乗せやすい
- 形態的・歴史的に現代主要言語とかなり離れており、難易度が高い

一方で、アッカド語にはシュメール語の表語文字が混ざることがあり、完全に単純な音写言語ではないとも説明されている。

## 使ったデータ

### データソース

データは主に以下のコーパスに由来する。

- ORACC
- CDLI
- ただし実験自体では、Frank Krueger が既に整形・収集したデータを使って既存研究と比較可能にしている

ORACC には恋愛文学、王碑文など複数のサブコーパスがあり、文体や反復性がかなり異なる。論文では、こうしたコーパス差が翻訳難易度やモデル挙動に影響することも示唆している。

### データ規模

論文中の設定は以下である。

- 総サンプル数: 95,629
- 学習用: 86,066
- テスト用: 9,563
- 分割: 90% train / 10% test
- 検証用 validation set: なし

validation を置かなかった理由は、ハイパーパラメータ探索を重視しておらず、比較実験を優先したためである。これは実験設計としてはやや粗いが、既存モデル比較という論文の目的には沿っている。

### データ上の重要な注意点

著者たちは、コーパスがかなり反復的で、学習集合とテスト集合に近い表現が多数現れると明言している。これは絶対スコアを高めに見せる可能性がある。つまり、

- この論文の点数は、より多様で現実的なデータでは下がる可能性がある
- ただし同一条件下でのモデル比較には依然として意味がある

## 比較したモデル

論文では 6 つの fine-tuned モデルを比較している。

### 1. T5-base

- 約 2.5 億パラメータ
- encoder-decoder 型
- 汎用 text-to-text モデル
- Krueger の先行研究でも使われており、基準線として採用

### 2. MarianMT Arabic-English

- 論文中では MarianAr
- 約 7500 万パラメータ
- encoder-decoder 型 NMT
- OPUS の並列コーパスで事前学習
- アラビア語と英語の翻訳経験を持つ

このモデルは、アッカド語と同じセム語派に属するアラビア語で事前学習されている点が最大のポイントである。著者たちは、この関連言語事前学習が非常に効いたと解釈している。

### 3. MarianMT Spanish-English

- 論文中では MarianEs
- MarianAr との対照実験
- スペイン語はアッカド語と系統的に遠い

これを比較対象に入れた狙いは、「翻訳モデルなら何でもよい」のか、それとも「関連言語で事前学習されていること」が重要なのかを確かめるためである。

### 4. Krueger の T5

- Frank Krueger の既存 T5 モデル
- アッカド語・シュメール語向けに既に学習済み
- 既存手法との比較ベースライン

### 5. Qwen 0.5B Instruct

- 約 5 億パラメータ
- decoder-only 型
- 多言語事前学習済み
- RoPE や Grouped-Query Attention など比較的新しい設計を採用

### 6. Mistral-7B

- 約 70 億パラメータ
- decoder-only 型
- few-shot 的な適応能力が期待される
- fine-tuning 時は LoRA と量子化を利用

Mistral-7B は消費 VRAM を抑えるため、Unsloth ライブラリを使い LoRA で訓練している。つまり全重み更新ではなく、追加の低ランク行列だけを学習している。

## どういう手法で訓練したのか

### 全体方針

この論文で採用している主手法は、一貫して fine-tuning である。RAG や外部検索、辞書制約付きデコードのような複雑な仕組みは中心ではない。

大まかな流れは次の通り。

1. アッカド語転写文と英訳の並列データを用意する
2. 既存の事前学習済み Transformer を選ぶ
3. 各モデルをアッカド語英訳タスクに fine-tune する
4. BLEU, ROUGE-2, BERTScore で比較する
5. 推論速度も測る

### モデル群の比較軸

著者たちは単純な精度比較ではなく、以下の軸を見ている。

- architecture の違い
  - seq2seq / encoder-decoder
  - causal / decoder-only
- 事前学習データの違い
  - 関連言語あり
  - 関連言語なし
- パラメータ数の違い
  - 軽量モデル
  - 大規模モデル
- 実用性の違い
  - 推論精度
  - 推論速度

### 学習設定

論文中で明示されている主要設定は次の通り。

- 学習率: 2e-5
- MarianMT 系と T5: 15 epoch
- Qwen 0.5B: 3 epoch
- Mistral-7B: 1 epoch
- validation set は使わない

epoch 数がモデルごとに異なるのは、計算資源の制約と、大規模モデルの few-shot / 少ない更新での適応力を見たかったためである。

### LoRA の役割

Mistral-7B の学習では VRAM 制約を回避するため LoRA を使っている。これは、

- 元モデルの大部分の重みを凍結する
- 小さな追加パラメータだけ学習する
- 省メモリで fine-tuning を行う

という方式である。論文では、LoRA によって消費資源を大きく下げつつ、十分な fine-tuning 性能が得られることを背景として説明している。

## 評価指標

論文は複数の観点から翻訳品質を測っている。

### BLEU

- 生成文と参照訳の n-gram 一致を見る
- 表層的一致を測る古典的指標

### ROUGE-2

- bigram 単位の precision / recall / F1 を測る
- 局所的なフレーズ構造の保存を見やすい

### BERTScore

- 文脈埋め込みベースで意味的近さを見る
- 単語が完全一致しなくても意味が近ければ高くなる

この 3 種類を併用することで、

- 単なる文字列一致
- フレーズ単位の一致
- 意味的な近さ

を分けて見ようとしている。

## 実験結果

## 全文対象の主結果

全テスト文での主要結果は以下である。

| モデル | BLEU | ROUGE-2 Prec | ROUGE-2 Recall | ROUGE-2 F1 | BERTScore |
| --- | ---: | ---: | ---: | ---: | ---: |
| Mistral-7B | 0.478 | 0.527 | 0.494 | 0.501 | 0.930 |
| MarianAr | 0.453 | 0.541 | 0.508 | 0.512 | 0.931 |
| Krueger T5 | 0.416 | 0.530 | 0.484 | 0.493 | 0.930 |
| Qwen 0.5B | 0.403 | 0.516 | 0.487 | 0.491 | 0.929 |
| T5 | 0.376 | 0.420 | 0.397 | 0.399 | 0.914 |
| MarianEs | 0.122 | 0.198 | 0.303 | 0.209 | 0.842 |

この結果から読み取れることは明快である。

- BLEU の最高は Mistral-7B
- ただし ROUGE-2 F1 と BERTScore は MarianAr が最良
- MarianEs は大差で悪い

つまり、

- 大規模 LLM は表層一致で強い
- しかし関連言語で事前学習された軽量 NMT は総合品質で非常に強い
- しかも非関連言語で事前学習した MarianEs は大きく崩れる

という構図が出ている。

## 長文と短文で傾向が違う

### 長文

参照訳が 4 語以上の長めの文では、

- Mistral-7B が BLEU で首位
- MarianAr が ROUGE と BERTScore で非常に強い
- 全体として MarianAr は安定して高い

長文では、単なる表層一致だけでなく、局所フレーズや意味保持で MarianAr が目立つ。

### 短文

参照訳が 4 語未満の短文では、

- Mistral-7B が BLEU 0.602
- Qwen 0.5B が BLEU 0.593
- Qwen が ROUGE-2 precision / recall / F1 と BERTScore の多くで上位

短文では LLM 群の few-shot 的な一般化能力が効きやすいと解釈されている。著者は、短いフレーズや定型表現では、大規模事前学習済み decoder-only モデルの語彙照合能力が有利だと見ている。

## 統計的有意差検定

論文は BLEU 差分に対して有意差検定も行っている。表ではモデル間の BLEU 差の信頼区間と p 値を示しており、すべての比較で p < 0.05 となっている。つまり、少なくともこのデータ条件では、各モデル間の BLEU の差は偶然ではなく統計的に有意と解釈している。

## 推論速度

500 文の翻訳に要した時間も比較している。

| モデル | 500文の総時間 | 1文あたり |
| --- | ---: | ---: |
| MarianAr | 108 秒 | 0.22 秒 |
| T5 | 242 秒 | 0.48 秒 |
| Qwen 0.5B | 348 秒 | 0.70 秒 |
| Mistral-7B | 1786 秒 | 3.57 秒 |

これはかなり重要な結果である。Mistral-7B は BLEU では最良だが、速度面では MarianAr に大きく負けている。実運用、特にリソース制約環境では、MarianAr のような軽量 NMT の方がはるかに実践的だと論文は結論づけている。

## 論文が主張している重要ポイント

### 1. 関連言語での事前学習は非常に効く

この論文で最も重要なのはここである。アラビア語とアッカド語は時代も表記も大きく違うが、どちらもセム語系である。そのため、Arabic-English で事前学習した MarianAr は、Spanish-English の MarianEs より圧倒的に良い結果を出した。

これは、

- 関連言語の構文や語彙的傾向
- 形態論的な類似
- 翻訳器が内部に持つ言語構造

が転移している可能性を示す。

著者たちは、「低資源言語向けには、関連言語で事前学習したモデルを探して fine-tune する」という戦略が有望だと示している。

### 2. LLM は強いが、常に最適ではない

Mistral-7B や Qwen は特に短文で強い。しかし、

- 学習・推論コストが高い
- 速度が遅い
- 総合バランスでは軽量翻訳モデルに劣る局面がある

つまり、「LLM があるから NMT は不要」という結論ではない。むしろこの論文は、低資源翻訳では小型の専門モデルが非常に強い場合があると示している。

### 3. アッカド語は低資源研究の良い試験場

アッカド語は、真に極端な低資源言語とは少し異なるが、

- データがある程度整備されている
- 古代語である
- 現代主要言語と距離がある
- 研究上意味が大きい

という条件を満たし、手法比較に向いていると論文は位置づけている。

## 論文の限界

著者自身が認めている限界もある。

### 1. データの反復性が高い

同じような表現が学習とテストに多数出てくるため、スコアが高めに出やすい。したがって、より多様なドメインでは結果が下がる可能性がある。

### 2. validation がなく、ハイパーパラメータ探索が限定的

厳密な最適化実験というより、モデル比較に重点を置いた設計である。そのため、各モデルの真の上限性能を出し切っているとは限らない。

### 3. アッカド語の条件は他の低資源言語より恵まれている

多くの低資源言語は、この論文のような高品質並列データを持たない。そのため、この設定で得られた知見を他言語へ移すには工夫が必要である。

### 4. OCR までは扱っていない

将来的には楔形文字画像から直接英訳する完全パイプラインが望まれるが、この論文はそこまでは扱っていない。

## 今後の展望として挙げられているもの

著者たちは今後の方向性として次を提案している。

- MarianAr や T5 をさらに長く学習し、追加 epoch やハイパーパラメータ探索を行う
- 軽量 NMT と重量級 LLM を組み合わせた mixture-of-experts 的構成を試す
- 楔形文字画像から OCR / 転写 / 翻訳までを一体化する
- 他の古代言語や低資源セム語にも同じ比較を広げる

## この論文を一言でまとめると

この論文は、

「低資源言語翻訳では、巨大 LLM をそのまま使うよりも、関連言語で事前学習された軽量翻訳モデルを丁寧に fine-tune した方が、品質と速度のバランスで優れることがある」

ことを、アッカド語を題材に具体的な比較実験で示した研究である。

## このリポジトリに引きつけて読むと重要な点

このリポジトリでアッカド語転写から英訳を扱うなら、論文から得られる実務上の示唆はかなり明確である。

- まずは seq2seq の翻訳モデルを基準線にする価値が高い
- 特に関連言語で事前学習されたモデルを優先的に試すべき
- BLEU だけでなく ROUGE-2 と意味類似も見た方が良い
- 短文と長文を分けて評価するとモデル特性が見えやすい
- 実運用を意識するなら推論速度の比較は必須

LLM を試す価値はあるが、少なくともこの論文の範囲では「軽量 NMT + 適切な事前学習言語選択」が非常に強い。
